# Security Policy

**Responsible disclosure guidelines for Arena**

</div>

---

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

| | |
|:--|:--|
| **Email** | security@playarena.app |
| **Subject** | Include "Security Vulnerability" in the subject line |
| **Details** | Provide as much information as possible |

### What to Include

| | |
|:--|:--|
| **Description** | Clear explanation of the vulnerability |
| **Steps** | How to reproduce the issue |
| **Impact** | Potential impact if exploited |
| **Suggestions** | Any ideas for fixing (optional) |

---

## Response Timeline

| Timeframe | Action |
|:--|:--|
| **24 hours** | Initial acknowledgment of your report |
| **72 hours** | Preliminary assessment |
| **7 days** | Detailed response with next steps |
| **30 days** | Target resolution for critical issues |

---

## Scope

### In Scope

| | |
|:--|:--|
| **Smart Contracts** | Any contract vulnerabilities |
| **Web Application** | XSS, CSRF, injection attacks |
| **API** | Authentication, authorization issues |
| **Wallet Integration** | Connection or signing vulnerabilities |
| **VRF Implementation** | Randomness manipulation |

### Out of Scope

| | |
|:--|:--|
| **Social Engineering** | Phishing attempts on team members |
| **DoS Attacks** | Denial of service without underlying vulnerability |
| **Third-Party Services** | Issues in Switchboard, Jupiter, or other dependencies |
| **Already Known** | Issues already reported or publicly known |

---

## Safe Harbor

We will not pursue legal action against researchers who:

| | |
|:--|:--|
| **Act in Good Faith** | Research conducted to improve security |
| **Avoid Harm** | No exploitation of vulnerabilities |
| **Report Promptly** | Disclose findings in a timely manner |
| **Keep Confidential** | Don't publicly disclose before fix is deployed |

---

## Security Measures

Arena implements multiple layers of security:

| Measure | Description |
|:--|:--|
| **Switchboard VRF** | Verifiable randomness for outcomes |
| **On-Chain Verification** | All transactions recorded on Solana |
| **Wallet Signatures** | Authentication via cryptographic signatures |
| **Rate Limiting** | Protection against abuse |
| **Input Validation** | Server-side validation of all inputs |

---

## Bug Bounty

We may offer rewards for significant security findings at our discretion. Rewards are based on:

| Factor | Consideration |
|:--|:--|
| **Severity** | Critical, High, Medium, Low |
| **Impact** | Potential damage if exploited |
| **Quality** | Clarity and completeness of report |
| **Originality** | First reporter of the issue |

---

## Contact

For security concerns: **security@playarena.app**

For general inquiries, please use our community channels.
